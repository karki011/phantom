## Phase 1: Hunter Stats Dashboard (High Priority)

**Goal:** Replace the placeholder stats with a real analytics dashboard powered by JSONL + stats-cache data.

### 1.1 — Activity Heatmap
- Read `~/.claude/stats-cache.json` (dailyActivity array)
- Render GitHub-style contribution heatmap (7 rows × 52 cols)
- Color scale using `--phantom-accent-*` tokens
- Show tooltip on hover: date, messages, sessions, tool calls

### 1.2 — Lifetime Stats Cards
- Total sessions, total tokens, estimated cost
- Favorite model (from DB `sessions.model` GROUP BY)
- Longest session, longest streak, current streak, active days
- Peak hour (from session timestamps)
- Source: PhantomOS DB (`sessions` table) + stats-cache

### 1.3 — Model Breakdown
- Pie/donut chart: sessions by model (opus/sonnet/haiku)
- Token usage by model
- Cost by model (using `estimated_cost_micros`)

### 1.4 — Session Timeline
- Scrollable timeline of recent sessions with duration bars
- Click to expand → see tasks, tokens, cost

### Data Sources
| Source | Path | Data |
|--------|------|------|
| Stats cache | `~/.claude/stats-cache.json` | Daily activity heatmap |
| PhantomOS DB | `phantom.db → sessions` | Tokens, costs, models, durations |
| PhantomOS DB | `phantom.db → tasks` | Task completion rates |

## Phase 3: Session Robustness (Medium Priority)

### 3.1 — SSE reconnection
- When the server SSE connection drops, auto-reconnect with backoff
- Show a "Reconnecting..." banner

### 3.2 — Session status detection
- Detect when a Claude Code session ends (process exit, JSONL stops growing)
- Auto-transition status from `active` to `completed`
- Currently sessions stay "active" forever unless manually ended

### 3.3 — JSONL scanner resilience
- Handle truncated/corrupt JSONL entries
- Handle file rotation (Claude Code can create new JSONL for same session after /clear)

## Phase 4: MCP Server Foundation (Low Priority — stretch)

### 4.1 — PhantomOS MCP server
